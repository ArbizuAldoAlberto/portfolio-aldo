#fecha: 2026-05-26

#tags: [memoria, lecciones-aprendidas, nextjs, docker]

Este archivo es la **fuente de la verdad operativa** del portafolio. Almacena las reglas del entorno, preferencias de sintaxis y soluciones a errores previos.

---

## 🛠️ Reglas del Entorno & Preferencias
*   **Syntax Rules:** React Server Components (RSC) por defecto en `/app`. Hooks y animaciones con `'use client'`. Zustand para el estado global y Framer Motion para animaciones.
*   **Estructura de Animación:** Mantener las animaciones limpias, fluidas, declarativas y controladas por GPU (como `transform` y `opacity`).
*   **Despliegue:** Docker-compose en VPS detrás de Nginx reverse proxy. El puerto del contenedor es el `3000` interno.

## 📖 Bitácora de Errores & Soluciones

### ❌ Error de tipado TS en Framer Motion ease arrays
*   **Fecha:** 2026-05-26
*   **Contexto:** Compilación de producción con Next.js/TypeScript.
*   **Causa Raíz:** TypeScript no infiere correctamente el array de números `[0.16, 1, 0.3, 1]` como una tupla fija de velocidad (`Easing`) esperada por Framer Motion, tratándolo como un array dinámico de números (`number[]`).
*   **Código Defectuoso:**
    ```typescript
    transition={{ duration: 0.8, ease: [0.16, 1, 0.3, 1] }}
    ```
*   **Solución Aplicada:**
    ```typescript
    transition={{ duration: 0.8, ease: [0.16, 1, 0.3, 1] as const }}
    ```
*   **Regla Preventiva:** Siempre declarar arrays de `ease` personalizados en transiciones de Framer Motion con la aserción `as const`.

### ❌ Git Clone fallido en VPS por repositorio privado
*   **Fecha:** 2026-05-26
*   **Contexto:** Despliegue continuo al VPS.
*   **Causa Raíz:** El repositorio de GitHub es privado y el VPS no cuenta con llaves SSH registradas ni credenciales configuradas en GitHub.
*   **Solución Aplicada:** Despliegue mediante tarball local empaquetado. Se comprime la carpeta excluyendo `node_modules` y `.next`, se transfiere por SCP y se descomprime en el VPS para luego reconstruir el contenedor con `docker compose build --no-cache`.
*   **Regla Preventiva:** Usar carga SCP de tarball empaquetado para despliegues rápidos sin exponer llaves del VPS a GitHub o viceversa.

### ❌ Comando `date` de PowerShell colisionando en SSH
*   **Fecha:** 2026-05-26
*   **Contexto:** Creación de backups automáticos desde la máquina local Windows por SSH.
*   **Causa Raíz:** Intentar correr `$(date +%Y%m%d)` en un comando SSH desde Windows PowerShell provoca un fallo porque PowerShell intercepta `date` localmente antes de mandarlo al SSH de Linux.
*   **Solución Aplicada:** Ejecutar comandos de fecha encapsulados directamente en comillas simples de Bash o automatizarlo del lado del servidor.

### 🌐 Arquitectura de Subdominios Multi-Tenant (El Dilema del Host)
*   **Fecha:** 2026-06-12
*   **Contexto:** Despliegue de 13 proyectos SaaS en subdominios (ej. `titanflow.aldoarbizu.com`).
*   **Causa Raíz:** El enrutador nativo de Next.js App Router y el middleware de Next-Intl no interceptan peticiones por subdominio directamente sin reescritura, lo cual dificulta la coexistencia de proyectos estáticos y dinámicos bajo el mismo ecosistema de Vercel/VPS.
*   **Solución Aplicada (Doble Capa):**
    1.  **Capa Local (Fallback):** En el archivo `middleware.ts`, implementamos una intercepción de `req.headers.get('host')`. Si detecta un subdominio válido, reescribe el pathname a `/sites/[subdomain]`.
    2.  **Capa Producción (Opus Magnus):** En el VPS, corremos `vhost_router.js` en el puerto 80 mediante PM2. Este router de Node.js puro es hiperveloz; ataja la petición, evalúa si es un subdominio estático (sirviendo los archivos HTML renderizados desde `/dashboards`) o si es dinámico (proxying puertos `3000`/`3020`). Si no es ninguno, recién ahí lanza el tráfico hacia el servidor local de Next.js (`3001`).
*   **Regla Preventiva:** Empaquetar siempre `vhost_router.js` junto con la carpeta de `dashboards/` como un bundle `hub-gateway.tar` para actualizaciones al nivel proxy.

### ✨ Glassmorphism Extremo (Founder Mode Walbi)
*   **Fecha:** 2026-06-12
*   **Contexto:** Mejorar el contraste visual y proyección de lujo (Wealth/Founder Mode) en la grilla de Proyectos.
*   **Regla Preventiva:** Para proyectos C-Level o Fintech B2B, no usar fondos planos. Emplear `backdrop-blur-2xl` y manipular `box-shadow: inset` con neones tenues (como el lila `#7C3AED` o cyan `#06B6D4`) reaccionando en hover para producir la estética HSL Liquid Glass.

### ❌ Cloudflare Zero Trust: Error 502 Bad Gateway
*   **Fecha:** 2026-06-19
*   **Contexto:** Exposición del puerto del Docker a través del servicio nativo `cloudflared`.
*   **Causa Raíz:** Cloudflare estaba configurado para buscar el tráfico de Next.js en `http://localhost:3001`, pero el contenedor interno de Next.js por defecto expone en el `3000`. Al enlazar el contenedor 3000:3000, el túnel no encontraba respuesta y arrojaba `502`.
*   **Solución Aplicada:** Engañar al túnel exponiendo el puerto interno `3000` de la app hacia el `3001` del VPS físico: `docker run -p 3001:3000`. 
*   **Regla Preventiva:** Nunca utilizar Nginx ni certificados locales (certbot) cuando se activa Cloudflare Tunnels para evitar redundancia y colisiones TLS. El despliegue de VPS B2B siempre va directo del puerto Docker al demonio `cloudflared`.

### 📊 Integración de APIs de Alta Frecuencia (Titan Flow)
*   **Fecha:** 2026-06-19
*   **Contexto:** El módulo de telemetría Titan Flow se quedaba colgado sin datos debido a limitación de APIs premium.
*   **Solución Aplicada:** Descartar APIs restrictivas. Migrar hacia `https://api.coingecko.com/api/v3/simple/price`.
*   **Regla Preventiva:** Siempre utilizar enfoques *Graceful Degradation* (degradación elegante) si la API falla o cae en límite 429. Cachear respuestas y evitar tumbar la renderización visual de React Three Fiber por una falla de conexión externa.

### 🎨 Renderizado de Texturas Críticas y Fallos Vercel (404)
*   **Fecha:** 2026-06-19
*   **Contexto:** Consola arrojando fallos 404 sobre archivos `noise.svg` y módulos `@vercel/analytics`.
*   **Solución Aplicada:** 
    1.  Si se aloja en VPS privado (Docker), arrancar de raíz todo módulo `@vercel/` en el `layout.tsx`.
    2.  No usar rutas URL para texturas críticas CSS. Codificarlas nativamente como `Base64 Data URI` (`url("data:image/svg+xml,...")`) para que vivan dentro del bundle JS y nunca arrojen 404 ni parpadeen al cargar.
*   **Regla Preventiva de UX Mobile:** Todo contenedor con múltiples columnas (ej. dashboards o paneles de métricas) debe declarar su resiliencia celular explícitamente: `grid-cols-1 sm:grid-cols-2`. Nunca forzar >2 columnas en móvil.

### 🔄 CI/CD Autónomo con Github Actions y VPS Docker
*   **Fecha:** 2026-06-22
*   **Contexto:** Necesidad de desplegar cambios continuamente a `aldoarbizu.com` sin entrar manualmente por SSH.
*   **Solución Aplicada:** Implementación de `.github/workflows/deploy.yml`. El script empaqueta el repo excluyendo `.git/node_modules`, lo sube vía SCP (usando llaves `VPS_SSH_KEY` inyectadas en Secrets), y ejecuta un script SSH remoto que hace `docker rm` y `docker build`.
*   **Regla Preventiva:** Para garantizar 0 fugas, el paso `scan_security_gate.js` se ejecuta ANTES del build local y el action de github. Si falla, se bloquea.

### 🌐 Multi-Locale y Chino Mandarín (`next-intl`)
*   **Fecha:** 2026-06-22
*   **Contexto:** Internacionalización a mercados asiáticos (Inglés, Español, Mandarín).
*   **Solución Aplicada:** Reestructuración con `messages/zh.json`. `toggleLanguage` cicla a través de `router.replace` inyectando el path `/[locale]`. Todo texto de componentes (Hero, Manifesto) extraído a `.json`.
*   **Regla Preventiva:** Ningún string *hardcodeado* en componentes React. Todo va a `useTranslations('Section')`.

### 📱 Experiencia Táctil (Swipe) y Modelos 3D Dinámicos
*   **Fecha:** 2026-06-22
*   **Contexto:** El usuario no descubría los 3 perfiles B2B (Engineer, Security, AgTech) en móvil. La batería del teléfono se drenaba por React Three Fiber.
*   **Solución Aplicada:**
    1.  **Swipe Táctil:** Inyectado `drag="x"` de `framer-motion` en el wrapper del componente central para ciclar perfiles.
    2.  **3D Contextual:** Uso de `useGLTF('/models/phone.glb')` para Engineer y `tree.glb` para el resto, precargados fuera del componente `Hero3D` para evitar destellos (FOUC).
    3.  **FPS Mobile:** Condicional `window.innerWidth < 768` para reducir la variable `particleCount` en `InteractiveStarRings` de 150 a 50.
